V TRACE - PAGE FUNCTIONALITIES DOCUMENTATION

================================================================================
PAGE 1: LOGIN PAGE (Authentication)
================================================================================

Functionality:
- User enters phone number
- System sends OTP via SMS
- User enters OTP to verify
- User gains access to the app

Features:
- Phone number input field
- OTP input field
- Send OTP button
- Verify OTP button
- Error messages for invalid inputs

Data Saved: Phone number (for session)


================================================================================
PAGE 2: EMPLOYEES DASHBOARD (Main Hub)
================================================================================

Functionality:
- Displays 5 module cards in a grid layout
- Each card opens a different module
- User can navigate between modules

Modules Available:
1. Visit Module
2. Travel Expense Calculator
3. Dynamic CRM
4. Crop Disease Report
5. Field Measurement

Features:
- Card-based interface
- Tap card to open module
- Back button to return to dashboard
- Module selection menu

Data Saved: None (Navigation only)


================================================================================
PAGE 3: VISIT MODULE
================================================================================

Functionality:
- Record customer/farmer visits
- Capture photos of location
- Add visit notes
- Save visit data to Firebase

Features:
- Customer name input field
- Visit notes/remarks text area
- Camera button to capture photo
- Photo preview
- Submit button to save
- Visit history display

Data Saved to Firebase:
- Date and time
- Customer name
- Visit notes
- Photo (as local file path)
- Unique visit ID


================================================================================
PAGE 4: TRAVEL EXPENSE CALCULATOR
================================================================================

Functionality:
- Track GPS location during travel
- Calculate distance traveled
- Calculate Travel Allowance (TA) at ₹5 per km
- Display real-time accuracy

Features:
- Start Tracking button
- Stop Tracking button
- Real-time distance display (in km)
- Real-time TA amount display (in ₹)
- Accuracy indicator (Green/Yellow/Red)
- Current location display
- Submit button to save expense
- Expense history display

Accuracy Levels:
- Green: <5m (Excellent)
- Yellow: 5-10m (Good)
- Red: >10m (Poor)

Data Saved to Firebase:
- Date and time
- Total distance (km)
- TA amount (₹)
- Start location (latitude, longitude)
- End location (latitude, longitude)
- Accuracy metrics
- Unique expense ID

================================================================================
PAGE 7: FIELD MEASUREMENT MODULE
================================================================================

Functionality:
- Measure field area using GPS
- Draw polygon on map by adding points
- Calculate area in square meters and hectares
- Display GPS accuracy in real-time

Features:
- Interactive map display
- Tap map to add points
- GPS button to add current location
- Real-time accuracy display (Green/Yellow/Red)
- Point counter
- Area calculation (automatic)
- Display area in m² and hectares
- Undo last point button
- Clear all points button
- Submit button to save measurement
- Measurement history display

Accuracy System:
- Each point takes 5 GPS readings (500ms intervals)
- Rejects readings with >10m accuracy
- Green: <5m accuracy
- Yellow: 5-10m accuracy
- Red: >10m accuracy

Data Saved to Firebase:
- Date and time
- All polygon points (latitude, longitude, accuracy)
- Total area (square meters)
- Total area (hectares)
- Quality metrics (average accuracy, max accuracy)
- Unique measurement ID


================================================================================
PAGE 8: TASKS MODULE
================================================================================

Functionality:
- Display daily tasks from Flask admin
- Accept or reject tasks
- Mark accepted tasks as completed or not completed
- Track task status and history

Task Status Flow:
Pending → Accept/Reject → Completed/Not Completed

Features:
- Pending tasks list
- Task details display
- Accept button
- Reject button (with reason input)
- Mark Completed button
- Mark Not Completed button (with reason input)
- Task priority display (High/Medium/Low)
- Overdue task indicator
- Task history display
- Summary statistics (total, completed, pending, rejected)

Priority Colors:
- Red: High priority
- Yellow: Medium priority
- Green: Low priority

Data Saved to Firebase:
- Task ID
- Status (pending/accepted/rejected/completed/not_completed)
- Rejection reason (if rejected)
- Incompletion reason (if not completed)
- Date and time of status change
- Task completion date

================================================================================
NAVIGATION STRUCTURE
================================================================================

Bottom Tab Navigation:
1. Employees Tab → Dashboard with 5 modules
2. Calendar Tab → History and activities view
3. Profile Tab → User profile and settings

Module Navigation:
- Dashboard → Select Module → Module Page → Submit Data → Back to Dashboard


================================================================================
DATA FLOW
================================================================================

All modules follow this flow:
1. User opens module from dashboard
2. User enters data/captures photos
3. User taps Submit button
4. Data saves to Firebase Realtime Database
5. Success message displayed
6. User can view history of saved data
7. Data syncs across all devices in real-time
